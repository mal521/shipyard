## Save Your Work

Your AI chat does not save automatically. If you close the tab or your session times out, that conversation is gone. Every time the AI gives you code you want to keep:

1. Copy it somewhere safe (a notes app, a Google Doc, or a code editor like VS Code)
2. Commit it to GitHub immediately
3. Or tell the AI: `"Give me a single file with everything we've built so far"`

Save early, save often. A closed tab is a lost session.

## The Builder Loop
```
1. Context      Tell AI about your situation and goals
2. Constraints  Tell it exactly what format and rules to follow
3. Iterate      Ask it to change, improve, and fix things
4. Ship         Put it live on the internet
```
This is how real products get built. Not in theory. You just did it from your laptop on a Zoom call.

## Prompt Templates

These are starting points. Change them, combine them, throw them out and write your own. The best projects usually come from your own ideas, not a template.

### Build a new app
```
I want to build a web app called [NAME].
What it does: [DESCRIPTION]
Requirements:
- Single HTML file with embedded CSS and JavaScript
- Clean, modern UI, mobile-friendly
- Use localStorage to save data between sessions
- Footer: "Built with AI by [YOUR NAME]"
Build the complete app.
```

### Add a project to your portfolio
```
Update my portfolio website to add a new project card:
- Project name: [NAME]
- Link: [URL]
- Description: [1-2 SENTENCES]
Keep the same design style as the existing cards.
```

### Fix something that's broken
```
Here's my code: [PASTE CODE]
I'm getting this error: [PASTE ERROR]
Fix it and explain what went wrong.
```
Pasting errors back into AI and asking it to fix them is called debugging. It's not a sign that something went wrong. It is the process. Professional developers do this hundreds of times a week.

### Go bigger
```
I have a working [DESCRIPTION OF CURRENT APP].
I want to add [NEW FEATURE].
Here's the current code: [PASTE CODE]
Add the feature without breaking what already works.
```
Dark mode toggle. Export to PDF. Data visualizations. Keyboard shortcuts. Keep pushing until something breaks, then fix it and push further. That's how you learn where the edges are.

## What To Do Next

1. **Tonight.** Send your site link to one person. A friend, a parent, a teacher. A live URL is more convincing than any description of what you built.
2. **This week.** Build one more app and add it to your portfolio. Pick something you'd actually use. A tool for studying, a tracker for a habit, a calculator for something specific to your life.
3. **This month.** Try building something that feels too ambitious. You will be surprised how far AI gets you. And the parts where it falls short are where you learn the most.
4. **Ongoing.** Every project you build goes on your portfolio. A portfolio with five working projects tells a different story than one with a single page. It compounds.
